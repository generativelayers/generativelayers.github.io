Getting Started

Install Generative Layers from Maven Central and start integrating LLMs into your BDI agents in minutes.

Install

Generative Layers is available on Maven Central. Add it to your project with one dependency.

<dependency>
    <groupId>com.generativelayers</groupId>
    <artifactId>generative-layers-core</artifactId>
    <version>0.2.0</version>
</dependency>
implementation 'com.generativelayers:generative-layers-core:0.2.0'
View on Maven Central | Source on GitHub

Prerequisites

Java 17+ JDK 17 or higher required.
Verify your Java version:
$ java -version
openjdk version "17.0.11" 2024-04-16
Download from Adoptium or Oracle JDK.
API Key From any supported provider — Cerebras, Groq, Gemini, OpenAI, or DeepSeek.
Set your key as an environment variable:
# Cerebras (free, no credit card)
$ export CEREBRAS_API_KEY="csk-...your-key-here"

# Groq (free tier: 30 req/min)
$ export GROQ_API_KEY="gsk_...your-key-here"

# Gemini (free tier: 15 req/min)
$ export GEMINI_API_KEY="AIzaSy...your-key-here"

# OpenAI
$ export OPENAI_API_KEY="sk-proj-...your-key-here"

# DeepSeek
$ export DEEPSEEK_API_KEY="sk-...your-key-here"
Get a free key from Cerebras, Groq, Google AI Studio, OpenAI, or DeepSeek. See Providers for full details.
Target Platform ASTRA, Jason, or JaCaMo.

What it looks like

A governed invocation with the v2 lifecycle. See the full lifecycle across all three platforms in the Platform syntax comparison.

// Jason — bind, call, validate, adopt
gl.bind("agent1", "gemini", "gemini-2.5-flash", "", Bid);
gl.call(Bid, "classify", "llm.answer", "ANSWER", "Classify: apple", "label,confidence", "", Rid);
gl.candidate(Rid, Cid);
gl.get(Cid, "label", Label);
gl.accept(Cid, "valid classification", Did);
.print("Accepted: ", Label).
What just happened?
  1. bind — bound the agent to Gemini as the active LLM provider.
  2. call — sent a prompt through the governance pipeline → provider → returned a result ID.
  3. candidate — isolated the LLM output as governed candidate material.
  4. get — extracted the label value from the validated JSON response.
  5. accept — promoted the candidate to accepted knowledge with a reason.
