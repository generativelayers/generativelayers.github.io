Research

Generative Layers investigates governed generative resource layers for agent-oriented systems.

Research motivation

Modern agentic AI systems increasingly use LLMs and external tools for planning, assessment, decision support, and action selection. In agent-oriented programming, this raises a semantic problem: generated material must not silently replace the host platform's reasoning model.

Generative Layers studies how external generative capabilities can be exposed as governed resources while preserving the semantics of beliefs, goals, intentions, plans, actions, and deliberation in the host agent system.

Problem statement

The central problem is how to let agent programs use LLMs, APIs, tools, services, and other external resources without allowing those outputs to automatically become beliefs, intentions, plans, or actions.

The framework therefore treats generated output as candidate external material that must pass through governance, policy checks, traceability, and explicit agent-side adoption.

Research questions

  1. How can generative model outputs be exposed to BDI agents without directly modifying beliefs, goals, intentions, or plans?
  2. What framework boundary is needed to govern, audit, and constrain generative resource access from agent-oriented programs?
  3. Can the same governed resource-layer model be applied across different agent platforms such as Jason and ASTRA?
  4. What are the trade-offs between placing generative capabilities inside the agent, inside the environment, or behind a separate governed resource layer?

Contribution

The project contributes a framework-oriented boundary between agent reasoning and external generative resource use, supporting policy checks, auditability, provider abstraction, and explicit agent-side adoption of generated material.

The implementation direction is a Java framework with platform-specific adapters for agent programming languages and MAS frameworks.

Scope

The initial scope is BDI-first and targets agent-oriented systems where reasoning semantics are explicit. Jason and ASTRA are the main early targets, with later investigation of JaCaMo and CArtAgO-style environments.

The project is not intended to replace an agent platform, hide LLM reasoning inside an agent cycle, or automatically convert generated text into operational beliefs or actions.

Methodology and evaluation plan

  • Define a common request, response, governance, and audit model for generative resource access.
  • Explore platform-specific adapters for established agent systems.
  • Compare direct LLM/tool usage against governed resource-layer usage.
  • Evaluate traceability, controllability, semantic separation, and cross-platform portability.
