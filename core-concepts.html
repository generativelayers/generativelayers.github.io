Core Concepts

The framework is built around explicit boundaries between reasoning, governance, and external resources.

Concept list

  • Controlled invocation: external capabilities are called through framework-defined requests.
  • Policy checks: requests and results can be accepted, denied, constrained, or flagged.
  • Auditable boundaries: interaction with external resources should be traceable.
  • Provider abstraction: LLMs, APIs, tools, and services are hidden behind provider interfaces.
  • Explicit adoption: generated material only affects the agent when the host platform adopts it.
  • Platform-specific adapters: each agent framework connects through a thin integration layer.

Design principle

The resource layer can suggest, assess, transform, or retrieve information. The agent layer remains responsible for reasoning and execution.
