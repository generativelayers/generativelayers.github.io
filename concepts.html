Concepts

Generative Layers separates agent reasoning from external generative resource use.

Conceptual layers

Agent layer

Host agent programs keep responsibility for goals, beliefs, plans, intentions, actions, and adoption decisions.

Platform adapter

A thin adapter connects the host agent language or framework to the shared Generative Layers request and response model.

Governance boundary

Policies, admissibility checks, logging, audit records, limits, and failure handling are applied before and after external resource access.

Provider abstraction

Providers connect governed requests to LLMs, APIs, tools, services, or assessment modules without exposing those resources directly to the agent.

Resource layer

External systems are treated as resources whose output is candidate material, not automatic agent state.

Key semantic rule

Generated material is not a belief, goal, intention, plan, or action merely because an external model produced it. The host agent platform must explicitly decide how to use it.
