Acceptance Patterns

Acceptance is up to the developer. The framework gives the agent governed candidate material; the agent decides whether to accept, reject, review, or refine it.

The Core Principle

The LLM is a tool, not a decision-maker. Output becomes candidate material, never automatically adopted belief. These examples demonstrate practical BDI-style governance: the agent validates, inspects, reviews, compares, or refines candidates before adoption.
