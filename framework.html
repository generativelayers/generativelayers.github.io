Framework

The framework boundary between agent programs and external generative resources.

Core concepts

Agent Layer

The host agent platform and its execution semantics.

  • Autonomous Control: Maintains the main BDI (Belief-Desire-Intention) reasoning loop, managing the agent internal state.
  • Asynchronous Triggers: Inquires external support asynchronously, maintaining internal stability while waiting for generative responses.
  • Context-Guarded Decision: Employs native agent guard conditions to decide whether to adopt, verify, ignore, or reject the candidate material.

Adapter

The bridge between an agent framework and Generative Layers.

  • Datatype Translation: Maps native agent datatypes, such as AgentSpeak terms, structures, and lists, to standard Java parameters and vice versa.
  • Syntactic Integration: Realizes the canonical API as native extensions, such as ASTRA Modules, Jason Internal Actions, or CArtAgO Artifacts.
  • Governed Lifecycle Binding: Creates isolated binding contexts per agent, each with its own governance kernel and body registry, while lifecycle stores remain globally shared so that any lifecycle ID is resolvable from any context.

Governance Boundary

Policy, limits, audit, and admissibility checks.

  • Decoupled Safety Policies: Enforces budget rules, token limits, rate caps, and content boundaries completely outside agent logic.
  • Assessment & Admissibility Gating: Candidates pass through an evidence-based assessment pipeline — evaluative verdicts recorded via judge() feed into an admissibility checker that gates whether accept() is permitted.
  • Audit Logging & Tracing: Automatically generates globally unique trace IDs to log request contexts, checked policies, and outbound results.
  • Pre-Adoption Verification: Intercepts responses, validating schemas and active rules before exposing candidate material to the agent.

Provider

The abstract connector to an LLM, tool, API, service, or module.

  • Resource Abstraction: Encapsulates model-specific API calls, tool setups, or service connections under a unified invocation contract.
  • Resilience & Networking: Isolates HTTP requests, handles rate limits, and manages retries or backoff without blocking agent threads.
  • Unified Interface: Simplifies provider switching, such as swapping model endpoints or mock providers, with zero changes to agent logic.

Candidate Material

External output that may be adopted, verified, ignored, or rejected.

  • Isolated State Container: Exposes generated values to the agent without automatically modifying internal beliefs or plans.
  • Lifecycle-Governed Status: Each candidate tracks its governance state through a lifecycle (VALIDATED → ASSESSED → ACCEPTED / REJECTED). State transitions gate which operations are valid — invalid candidates cannot be assessed, and final states cannot be reversed.
  • Structured Inspections: Provides safe query boundaries through check() and get() so agents can inspect data before adoption.
  • Deliberated Incorporation: Candidate fields become GL-side knowledge only when the agent explicitly adopts the candidate through accept(candidateId, reason).

Request path

1. see() discovers available providers. bind() creates an isolated agent–provider context.
Discovery & Binding
↓
2. call() performs one governed invocation: policy gate → provider → validation → candidate creation.
Invocation
↓
3. result() inspects the outcome. candidate() crosses into governed material. check() and get() query content.
Inspection
↓
4. judge() records evaluative evidence. decide() computes admissibility from assessment verdicts.
Assessment
↓
5. accept() adopts admissible material. reject() discards it. Both are terminal — no reversal.
Decision
↓
6. knowledge() retrieves accepted GL-side knowledge. explain() audits any lifecycle object.

Canonical commands

The framework establishes a strict canonical contract of 13 lifecycle commands that behave identically across all target platforms. The contract is defined in the Java ResourceActions interface, ensuring full functional parity across integrations.
