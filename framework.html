Framework

The framework boundary between agent programs and external generative resources.

Request path

1. Agent asks for external support.
Reasoning
↓
2. Adapter creates a Generative Layers request.
Adapter
↓
3. Governance checks policy, limits, context, and audit requirements.
Governance
↓
4. Provider calls an LLM, tool, API, service, or module.
Provider
↓
5. Result returns as candidate material.
Material
↓
6. Agent decides whether to adopt, verify, ignore, or reject it.
Deliberation

Core concepts

Agent layer

The host agent platform and its reasoning semantics.

Adapter

The bridge between an agent framework and Generative Layers.

Governance boundary

Policy, limits, audit, and admissibility checks.

Provider

The connector to an LLM, tool, API, service, or module.

Candidate material

External output that may be adopted, verified, ignored, or rejected.

Canonical commands

The framework establishes a strict canonical contract of 16 core commands that behave identically across all target platforms. The contract is defined in the Java ResourceActions interface, ensuring full functional parity across integrations.

Command Return Type Description
configure(key, value)
boolean Set a configuration key-value pair prior to provider activation.
Syntax & Usage Examples 
// ASTRA
gl.configure("model", "gemini-1.5-flash");
// Jason
gl.configure("model", "gemini-1.5-flash");
// JaCaMo
configure("model", "gemini-1.5-flash");
use_provider() (+2 overloads)
boolean Activate a generative provider (by default, or named with optional model).
Syntax & Usage Examples 
// ASTRA

// Jason

// JaCaMo

providers()
String Fetch a comma-separated list of all available provider names.
Syntax & Usage Examples 
// ASTRA

// Jason

// JaCaMo

invoke(agent, goal, body, affordance, prompt, requiredCsv)
String Execute a defined generative body (affordance model interaction). Returns result ID.
Syntax & Usage Examples

// Jason

// JaCaMo
invoke("agent_a", "classify", "llm.answer", "ANSWER", "Classify", "label", Rid);

Platform syntax comparison

Compare how the exact same core flow (configuring, invoking, validating, and adopting candidate material) is written natively across the target agent languages:

ASTRA PLATFORM
module gl.adapter.astra.AstraAdapter gl;
 
 rule +!main(list args) {
     // 1. Setup provider
     gl.configure("model", "gemini-1.5-flash");
     gl.use_provider("gemini");
     
     // 2. Trigger asynchronous query sub-goal
     !request_support("my_agent", "solve_task", "Create candidate plan");
 }
 
 // Subgoal plan: invokes the resource asynchronously
 rule +!request_support(string agent, string goal, string prompt) {
     string rid = gl.ask(agent, goal, prompt);
     !deliberate_result(rid);
 }
 
 // Deliberation subgoal plans using pattern-matching guards:
 rule +!deliberate_result(string rid) : gl.valid(rid) == true {
     string cid = gl.candidate(rid);
     !deliberate_candidate(cid);
 }
 
 rule +!deliberate_result(string rid) : gl.valid(rid) == false {
     console.println("Generated output is invalid: " + gl.outcome(rid));
 }
 
 // Deliberating candidate with BDI context guards:
 rule +!deliberate_candidate(string cid) : gl.admissible(cid) == true {
     gl.accept(cid);
 }
 
 rule +!deliberate_candidate(string cid) : gl.admissible(cid) == false {
     gl.reject(cid);
 }
!main.
 
 +!main <-
     // 1. Setup provider
     gl.configure("model", "gemini-1.5-flash");
     gl.use_provider("gemini");
     
     // 2. Trigger asynchronous query sub-goal
     !request_support("my_agent", "solve_task", "Create candidate plan").
 
 +!request_support(Agent, Goal, Prompt) <-
     gl.ask(Agent, Goal, Prompt, Rid);
     !deliberate_result(Rid).
 
 // Deliberation subgoals reacting to context guards instead of if-else:
 +!deliberate_result(Rid)
     :   gl.valid(Rid, true)
     <-  gl.candidate(Rid, Cid);
         !deliberate_candidate(Cid).
 
 +!deliberate_result(Rid)
     :   gl.valid(Rid, false)
     <-  gl.outcome(Rid, Outcome);
         .print("Generated output is invalid: ", Outcome).
 
 // Deliberating candidate with BDI guard conditions:
 +!deliberate_candidate(Cid)
     :   gl.admissible(Cid, true)
     <-  gl.accept(Cid).
 
 +!deliberate_candidate(Cid)
     :   gl.admissible(Cid, false)
     <-  gl.reject(Cid).
!main.
 
 +!main <-
     // 1. Focus on the CArtAgO environment artifact
     makeArtifact("gl", "gl.adapter.jacamo.JaCaMoAdapter", [], Id);
     focus(Id);
     
     // 2. Configure and invoke standard operations
     configure("model", "gemini-1.5-flash");
     use_provider("gemini");
     ask("agent", "solve_task", "Create candidate plan", Rid).
 
 // 3. Reacting to CArtAgO observable belief updates asynchronously:
 +gl_result(Rid, Outcome, true) <-
     !deliberate_result(Rid).
 
 +gl_result(Rid, Outcome, false) <-
     .print("Generated output is invalid: ", Outcome).
 
 // Subgoal delegation:
 +!deliberate_result(Rid) <-
     candidate(Rid, Cid);
     !check_admissibility(Cid).
 
 // BDI Guard conditions on sub-plans:
 +!check_admissibility(Cid)
     :   admissible(Cid, true)
     <-  accept(Cid).
 
 +!check_admissibility(Cid)
     :   admissible(Cid, false)
     <-  reject(Cid).

Design rules

  • Do not silently insert beliefs.
  • Do not silently execute generated tool calls.
  • Expose denials, warnings, and failures.
  • Keep adapters thin.
  • Keep the host agent language in control.
