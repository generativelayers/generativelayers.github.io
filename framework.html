Core concepts

Agent Layer

The host agent platform and its execution semantics.

  • Autonomous Control: Maintains the main BDI (Belief-Desire-Intention) reasoning loop, managing the agent's internal state.
  • Asynchronous Triggers: Inquires external support asynchronously, maintaining internal stability while waiting for generative responses.
  • Context-Guarded Decision: Employs native agent guard conditions to decide whether to adopt, verify, ignore, or reject the candidate material.

Adapter

The bridge between an agent framework and Generative Layers.

  • Datatype Translation: Maps native agent datatypes (like AgentSpeak terms, structures, and lists) to standard Java parameters and vice versa.
  • Syntactic Integration: Realizes the canonical API as native extensions, such as ASTRA Modules, Jason Internal Actions, or CArtAgO Artifacts.
  • Asynchronous Lifecycle Binding: Manages request session tracking, binding asynchronous responses to the initiating agent context.

Governance Boundary

Policy, limits, audit, and admissibility checks.

  • Decoupled Safety Policies: Enforces budget rules, token limits, rate caps, and content boundaries completely outside of agent logic.
  • Audit Logging & Tracing: Automatically generates globally unique trace IDs to log all request contexts, policies checked, and outbound results.
  • Pre-Adoption Verification: Intercepts responses, validating schemas and active rules before exposing candidate material to the agent.

Provider

The abstract connector to an LLM, tool, API, service, or module.

  • Resource Abstraction: Encapsulates model-specific API calls, tool setups, or service connections under a unified invocation contract.
  • Resilience & Networking: Isolates HTTP requests, handles rate limits, and manages retries/backoffs without blocking agent threads.
  • Unified Interface: Simplifies provider switching (e.g. swapping model endpoints or mock providers) with zero changes to agent logic.

Candidate Material

External output that may be adopted, verified, ignored, or rejected.

  • Isolated State Container: Exposes the generated values to the agent without automatically modifying any internal beliefs or plans.
  • Structured Inspections: Provides safe query boundaries (via valid() and field()) so agents can inspect the data before adoption.
  • Deliberated Incorporation: Merges fields into the agent's belief base only when the agent explicitly adopts the candidate (via accept()).

Canonical commands

The framework establishes a strict canonical contract of 16 core commands that behave identically across all target platforms. The contract is defined in the Java ResourceActions interface, ensuring full functional parity across integrations.
