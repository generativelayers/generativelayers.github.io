The framework boundary between agent programs and external generative resources.

Core concepts

Agent layer

The host agent platform and its reasoning semantics.

Adapter

The bridge between an agent framework and Generative Layers.

Governance boundary

Policy, limits, audit, and admissibility checks.

Provider

The connector to an LLM, tool, API, service, or module.

Candidate material

External output that may be adopted, verified, ignored, or rejected.

Canonical commands

The framework establishes a strict canonical contract of 16 core commands that behave identically across all target platforms. The contract is defined in the Java ResourceActions interface, ensuring full functional parity across integrations.

Command Return Type Description
configure(key, value) boolean Set a configuration key-value pair prior to provider activation.
use_provider() (+2 overloads) boolean Activate a generative provider (by default, or named with optional model).
providers() String Fetch a comma-separated list of all available provider names.
invoke(agent, goal, body, affordance, prompt, requiredCsv) String Execute a defined generative body (affordance model interaction). Returns result ID.
invoke_with_beliefs(...) String Execute a generative body, prepending structured beliefs as RAG context.
ask(agentId, goalId, prompt) String Shorthand LLM invocation using the baseline standard answer affordance.
valid(resultId) boolean Check whether the generated output passed schema validation constraint checks.
field(resultId, name) String Extract a specific named field's value from a validated response schema.
candidate(resultId) String Retrieve the unique Candidate ID associated with a result.
trace(resultId) String Get the auditability trace ID generated for the invocation lifecycle.
outcome(resultId) String Get the outcome name (e.g. VALIDATED, INVALID, GOVERNANCE_DENIED).
knowledge(agentId) String Retrieve all accepted knowledge formatted as semicolon-separated key-value entries.
admissible(candidateId) boolean Check whether the candidate currently satisfies all active governance policies.
accept(candidateId) boolean Explicit adoption of the candidate, incorporating its fields into the agent's beliefs.
reject(candidateId) boolean Reject a candidate output, refusing adoption.
assess(...) boolean Record a peer verdict assessment of another agent's candidate.

Platform syntax comparison

Compare how the exact same core flow (configuring, invoking, validating, and adopting candidate material) is written natively across the target agent languages:

ASTRA PLATFORM
module gl.adapter.astra.AstraAdapter gl;

rule +!main(list args) {
    // 1. Configure and use provider
    gl.configure("model", "gemini-1.5-flash");
    gl.use_provider("gemini");
    
    // 2. Invoke and check validity
    string rid = gl.ask("agent", "goal", "prompt");
    if (gl.valid(rid)) {
        string cid = gl.candidate(rid);
        
        // 3. Deliberate and adopt
        if (gl.admissible(cid)) {
            gl.accept(cid);
        } else {
            gl.reject(cid);
        }
    }
}
!main.

+!main <-
    // 1. Configure and use provider
    gl.actions.configure("model", "gemini-1.5-flash", _);
    gl.actions.use_provider("gemini", _);
    
    // 2. Invoke and check validity
    gl.actions.ask("agent", "goal", "prompt", Rid);
    gl.actions.valid(Rid, Valid);
    
    if (Valid == true) {
        gl.actions.candidate(Rid, Cid);
        
        // 3. Deliberate and adopt
        gl.actions.admissible(Cid, Admissible);
        if (Admissible == true) {
            gl.actions.accept(Cid, _);
        } else {
            gl.actions.reject(Cid, _);
        }
    }.
!main.

+!main <-
    // 1. Focus on the CArtAgO environment artifact
    makeArtifact("gl", "gl.adapter.jacamo.JaCaMoAdapter", [], Id);
    focus(Id);
    
    // 2. Configure and invoke using standard operations
    configure("model", "gemini-1.5-flash");
    use_provider("gemini");
    ask("agent", "goal", "prompt", Rid).

// 3. Deliberation triggered automatically by sensing observable property gl_result
+gl_result(Rid, Outcome, true) <-
    candidate(Rid, Cid);
    admissible(Cid, true);
    accept(Cid).

+gl_result(Rid, Outcome, false) <-
    .print("Generated output is invalid").

Design rules

  • Do not silently insert beliefs.
  • Do not silently execute generated tool calls.
  • Expose denials, warnings, and failures.
  • Keep adapters thin.
  • Keep the host agent language in control.
