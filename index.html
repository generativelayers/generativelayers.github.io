Welcome to Generative Layers

Generative Layers is a framework direction for building governed generative agents with controlled, auditable interaction boundaries for external models, tools, services, and resources.

View GitHub Read overview

Introduction and Overview

The project focuses on separating agent decision logic from external generative capabilities. Instead of allowing an external model or tool to become an uncontrolled part of the agent's reasoning process, Generative Layers places those capabilities behind explicit interaction boundaries.

The intended result is a reusable framework for agent-oriented systems where calls to models, APIs, tools, and resources can be governed, inspected, and adapted without destroying the agent structure around them.

Generative Layers Concepts

Agent layer

Represents the agent-side goals, decisions, plans, runtime context, and execution flow.

Governance layer

Checks whether an interaction is allowed, records failures, and emits audit information.

Resource layer

Provides controlled access to LLMs, APIs, tools, services, and external systems.

Core Framework Concepts

  • Controlled invocation: external calls pass through explicit framework interfaces.
  • Policy checks: interactions can be allowed, denied, logged, or escalated.
  • Auditable boundaries: prompts, responses, tools, decisions, and violations can be traced.
  • Provider abstraction: external services can be swapped without changing the agent-facing model.
