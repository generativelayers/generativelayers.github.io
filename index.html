Early research framework

Generative Layers

A Java framework for governed generative resource layers around agent-oriented systems.

Purpose

Generative Layers keeps external generation separate from the agent's core execution loop.

LLM, tool, API, and service outputs are returned as candidate material. The host agent platform decides whether to adopt, verify, ignore, or reject them.

Target platforms

ASTRA

AgentSpeak(TR)-style BDI target.

Jason

AgentSpeak-style BDI target.

JaCaMo

Jason, CArtAgO, and Moise integration.

Integration Details & Specifications

Governance architecture, target platforms, and custom integration adapters.

Generative Layers is built to support secure integration with prominent BDI (Belief-Desire-Intention) agent languages and multi-agent programming platforms. The core framework currently offers native, production-tested integrations for ASTRA, Jason, and the integrated JaCaMo platform (which binds Jason BDI agents, CArtAgO environments, and Moise organizations together).

Compare how the same governance flow (configuring, invoking, validating, and adopting candidate material) is realized natively across each target agent environment:

ASTRA Module

Implemented as a native ASTRA Module written in Java. Commands are mapped to inline custom statements prefixed with the module binding gl.

Jason Internal Actions

Implemented as standard AgentSpeak Internal Actions in Java. Invoked natively by BDI agents using custom extensions prefixed with gl..

CArtAgO Artifact (JaCaMo)

Implemented as a CArtAgO Environment Artifact in Java. Commands are operations called once focused on the artifact, receiving updates via observable beliefs.
